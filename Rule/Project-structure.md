## 🏗️ Architectural Layers

### **Entry Points Layer** (User-Facing)
- `RDVH.ps1` - Complete RDS Virtual Desktop Infrastructure
- `RDS.ps1` - RDS Session Host farm
- `Domainsetup.ps1` - Domain environment
- `deploy.ps1` - Basic VM deployment

### **Orchestration Layer** (Workflow Management)
- High-level script coordination
- Parameter validation and conversion
- Progress reporting and error handling
- Replay command generation

### **Core Engines Layer** (Building Blocks)
- `deploy.ps1` - VM instantiation from golden images
- `createDC.ps1` - Domain controller provisioning
- `joindomain.ps1` - Domain membership management

### **Foundation Layer** (Infrastructure)
- `setup.ps1` - Image preparation and downloads
- `switch.ps1` - Network configuration
- `unattend.ps1` - Image customization

### **Utility Layer** (Supporting Tools)
- `cleanup.ps1` - Resource management
- `DHCP.ps1` - Network services
- Configuration files and logs

## 📋 File Categories

### **Script Files** (`.ps1`)
- **Orchestrators**: End-to-end automation workflows
- **Engines**: Core functionality implementation
- **Utilities**: Specialized operations and maintenance

### **Configuration Files**
- `sys_bootstrap.ini` - Master password (required by all scripts)
- `switch.txt` - Network settings (generated by switch.ps1)

### **Generated Assets**
- `goldenImage/` - Base VHD images for VM deployment

### **Documentation**
- `readme.txt` - Technical documentation
- `Guidance.txt` - User guidance and examples

## 📊 Dependency Relationships

### **Direct Dependencies** (Script A calls Script B)
- `RDVH.ps1` → `createDC.ps1`, `deploy.ps1`, `joindomain.ps1`
- `RDS.ps1` → `createDC.ps1`, `deploy.ps1`, `joindomain.ps1`
- `Domainsetup.ps1` → `createDC.ps1`, `deploy.ps1`, `joindomain.ps1`
- `setup.ps1` → calls `switch.ps1` when needed

### **File Dependencies**
- **All Scripts**: Require `sys_bootstrap.ini`
- **deploy.ps1**: Requires `switch.txt`, `goldenImage/*.vhd`
- **setup.ps1**: Creates `goldenImage/`, may create `switch.txt`

### **Execution Order Constraints**
1. Network setup (switch.ps1) before VM deployment
2. Image preparation (setup.ps1) before VM creation
3. VM deployment before domain operations
4. Domain controller before domain joins

## 🎯 Design Principles

### **Separation of Concerns**
- Each script has a single, well-defined responsibility
- Orchestrators coordinate, engines execute, utilities support

### **Layered Architecture**
- Clear dependency hierarchy prevents circular references
- Higher layers depend on lower layers, not vice versa

### **Configuration Management**
- Centralized configuration files
- Generated assets separated from source code
- Logs isolated for debugging

### **Error Resilience**
- Comprehensive error handling at all levels
- Replay commands for failure recovery
- Idempotent operations where possible

## 🚀 Scaling Considerations

### **Modular Design**
- Scripts can be used independently or composed
- New entry points can reuse existing engines
- Utilities can be added without affecting core workflows

### **Asset Management**
- Golden images enable rapid VM deployment
- Centralized logging for troubleshooting
- Clean separation of generated vs. source files

### **Maintenance**
- Clear file organization simplifies updates
- Documentation in dedicated folders
- Standards in Rule/ directory for consistency
